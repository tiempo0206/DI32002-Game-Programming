## 5.9 Class Activity

### What the game did before

- The level had a playable MVP scene with movement, shooting, enemy spawning, score, win/lose flow, background art, and a simple start overlay.
- The start flow was incomplete for the assignment rubric because it did not include a full instructions page, return flow, polished menu audio, or a stronger in-game HUD.

### What was updated today

- Rebuilt the start flow into a classic arcade-style menu with `NEW GAME`, `INSTRUCTIONS`, `BACK`, and `EXIT`.
- Added looping menu music and separated it from gameplay music.
- Added score, high score, HP, and target progress to the HUD.
- Added clear in-game objective text so the player knows the goal immediately.
- Increased player movement speed and expanded the playable area by pushing the asteroid boundary and camera range outward.
- Added score-based difficulty scaling so enemy pressure increases during a run.
- Added gameplay audio feedback for player fire, enemy fire, player damage, enemy damage, victory, and game over.
- Added retry and menu-return buttons to both the victory and game-over pages.

### Why these changes improve the player experience

- The player now understands how to start, control, and finish the game without guessing.
- The HUD gives immediate feedback about survival and progress.
- Audio and objective text make actions and results easier to read.
- Faster movement, a larger arena, and scaling enemy pressure make the loop feel more intentional and dynamic.

### Test checklist

- Open the scene and confirm the menu appears first.
- Open `INSTRUCTIONS`, then return with `BACK`.
- Start the game and confirm movement, aiming, and shooting work.
- Confirm score increases when enemies are defeated.
- Confirm HP changes when the player is damaged.
- Confirm target progress updates toward the win condition.
- Confirm the victory page and game-over page both allow retry and return to menu.
